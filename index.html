Chapter 01 — Evidence Collection

The Alley

console.log · let · const

The rain hadn't let up in three days. Detective Sprout ducked under the yellow tape and stepped into the street where Victor Ember had been found face-down in his own blood. The first rule of any investigation is simple: write everything down. Every detail matters, even the ones that seem small. You don't solve a case from memory you solve it from evidence.

In JavaScript, a variable is your notepad. const is like a photograph sealed in an evidence bag. Once it's in there, it doesn't change. let is a sticky note you can update. And console.log() is your flashlight — it opens the notepad and reads out what's inside.

// const — sealed evidence, never changes: const victimName = "Victor Ember"; // let — may change as the case develops: let suspectCount = 4; let timeOfDeath = "11:42 PM"; // console.log reads it back to you: console.log(victimName); // "Victor Ember" console.log(suspectCount); // 4 // As the case develops, update a let: suspectCount = 3; console.log(suspectCount); // 3 — one cleared

addEventListener · classList

Sprout lined up the suspects in the cold fluorescent light. Each one gave nothing away. Until you applied pressure. A good detective doesn't just observe the room, he creates reactions and reads them. JavaScript works the same way. An event listener watches for something to happen, a click, a keypress, and then runs a piece of code in response.

When a reaction occurs, you often need to visibly change something on the page. That's classList. With classList.toggle(), JavaScript can swap CSS classes in and out like swapping a suspect's expression under pressure, revealing what was always underneath.

const suspect = document.querySelector( '.suspect-card' ); // Watch for a click — like slamming // a folder on the table: suspect.addEventListener('click', function() { // Toggle 'nervous' class on/off: suspect.classList.toggle('nervous'); } ); // In your CSS, .nervous might turn // the card red or shake it.

Chapter 04 — The Darkroom

The
Darkroom

CSS Custom Properties

In the basement of the precinct, Sprout hung the photographs to develop. In the red glow of the safelight, shapes emerged slowly from blank white paper — details invisible in the chaos of the crime scene suddenly made sharp and undeniable. A darkroom doesn't create new information; it reveals what was always there.

CSS custom properties — called CSS variables — work like the controls in that darkroom. You define them once using -- as a prefix, then use them throughout your CSS. The real power arrives when JavaScript reaches in and changes those variables directly. Change one, and every element using it updates instantly, like turning off the light in the darkroom.

/* Define your variables once: */ :root { --scene-bg: #0a0818; --accent-color: #3d7fff; } /* Use them in CSS: */ body { background: var(--scene-bg); } // JS changes a variable at runtime: document.documentElement .style.setProperty( '--scene-bg', '#080005' ); // Read it back: getComputedStyle(root) .getPropertyValue('--scene-bg'); // → '#080005' // Every element using that // variable updates immediately.

THE FILING CABINET: localStorage

What it is: localStorage is a key-value store built into every browser. Unlike regular JavaScript variables — which disappear when the page closes — localStorage data persists on the user's device indefinitely, until the user clears their browser data or you explicitly delete it.

Why it matters: localStorage lets you build experiences that remember users — saved settings, game progress, shopping carts, form drafts, dark/light mode preferences. It holds up to ~5MB per domain and needs no server or login.

The catch: Everything stored is a string. To store objects or arrays, use JSON.stringify() to write and JSON.parse() to read them back.

// Save evidence to the case file:
localStorage.setItem('clues', cluesFound);
localStorage.setItem('suspect', 'The Butler');
// Retrieve it later (even after reload):
localStorage.getItem('clues'); // → "3"
// Store an object — convert first:
localStorage.setItem('case',
  JSON.stringify({ clues: 3, confirmed: true }));
// Read it back as an object:
const notes = JSON.parse(
  localStorage.getItem('case'));
// Remove a key:
localStorage.removeItem('clues');
