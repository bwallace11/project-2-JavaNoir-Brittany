Chapter 01 — First Contact

The Alley

console.log · let · const

First Contact — console.log & DevTools. Before you investigate, you need a way to see what’s happening. Open DevTools (press F12) and go to the Console tab. That’s where console.log() sends messages. It’s your flashlight into the code. Every clue you click on this page logs a message there. You write a line, and the machine talks back.

// First Contact — open DevTools (F12): console.log("Case opened."); console.log("Victim:", "Victor Ember"); console.log("Clues found:", 0); // Check the Console tab — it replies.

Naming Things — let & const. Now that you can see the output, you need somewhere to store facts. In JavaScript, a variable is your notepad. const is like a photograph sealed in an evidence bag. Once it’s there, it doesn’t change. let is a sticky note you can update as the case unfolds. Name your evidence, lock down what’s certain, and keep the rest flexible.

// Naming Things — let vs const: const victimName = "Victor Ember"; let suspectCount = 4; // const can't be reassigned: // victimName = "Jane"; // ERROR! // let can change as the case develops: suspectCount = 3; console.log(suspectCount); // 3 — one cleared

Chapter 02 — The Interrogation Room

The Room
Reacts

addEventListener · classList · conditionals

Listening. Sprout lined up the suspects under cold fluorescent light. They gave nothing away until pressure was applied. A good detective creates reactions and reads them. In JavaScript, an event listener watches for something to happen like a click, keypress, or hover. When it happens, code runs. Without listeners, a page just sits there. With them, it listens.

// Listening — addEventListener: const lamp = document.querySelector( '#lamp-toggle-btn' ); // Watch for a click: lamp.addEventListener('click', function() { console.log('Light toggled.'); } ); // Now every click runs that code.

When a reaction happens, you make a decision. That is a conditional using if/else. It checks a condition and runs different code depending on the result. With classList, you can change CSS classes to update what the page shows. classList.add() pins a label on. classList.remove() tears one off. classList.toggle() flips it. classList.contains() checks if it is there.

// The Switch — conditionals + classList: const card = document.querySelector( '.suspect-card' ); card.addEventListener('click', () => { card.classList.toggle('interrogated'); // The conditional — check and decide: if (card.classList.contains( 'interrogated')) { console.log('Story changes.'); } else { console.log('Still hiding.'); } });

Chapter 04 — The System

The
Darkroom

CSS Custom Properties · Design Tokens

In the basement of the precinct, Sprout hung photographs to develop. Under the red safelight, shapes slowly appeared from blank paper. Details hidden at the crime scene became clear. A darkroom does not create information. It reveals what was already there. The detective controls the process with two dials: time in the chemicals and the intensity of the light. Turn one dial and the whole room changes.

CSS custom properties, also called CSS variables, work the same way. You define them once using --, then reference them throughout your CSS with var(). JavaScript can change those variables while the page is running. Change one value and every element using it updates instantly. Toggle the switch above and watch the scene transform: background, glow, and color temperature all change from just two custom properties.

/* The System — define tokens once: */ :root { --scene-bg: #0a0818; --accent-color: #3d7fff; } /* Every element reads from them: */ body { background: var(--scene-bg); } // JS changes a variable at runtime: document.documentElement .style.setProperty( '--scene-bg', '#080005' ); // Read it back: getComputedStyle(root) .getPropertyValue('--scene-bg'); // → '#080005' // One source of truth. Change it, // and everything updates instantly.

THE FILING CABINET: localStorage

What it is: localStorage is a key value store built into every browser. Regular JavaScript variables disappear when the page closes. localStorage data stays on the user's device until it is cleared or deleted.

Why it matters: localStorage lets sites remember users. It can store saved settings, game progress, shopping carts, form drafts, or dark and light mode preferences. It holds about 5MB per domain and needs no server.

The catch: Everything stored is a string. To store objects or arrays use JSON.stringify() to save them and JSON.parse() to read them back.

// Save evidence to the case file:
localStorage.setItem('clues', cluesFound);
localStorage.setItem('suspect', 'The Butler');
// Retrieve it later (even after reload):
localStorage.getItem('clues'); // → "3"
// Store an object — convert first:
localStorage.setItem('case',
  JSON.stringify({ clues: 3, confirmed: true }));
// Read it back as an object:
const notes = JSON.parse(
  localStorage.getItem('case'));
// Remove a key:
localStorage.removeItem('clues');
